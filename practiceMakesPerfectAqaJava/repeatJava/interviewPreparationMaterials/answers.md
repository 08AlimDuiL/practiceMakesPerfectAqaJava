### Перегрузка и переопределение

Что такое перегрузка и переопределение методов?

Два разных способа изменения методов в Java.

1. В Java есть возможность в одном и том же классе определять два или более метода с одним и тем же именем, если только
   объявления их входных параметров отличаются. Это называется **перегрузкой методов**(**Overloading**). Данные механизм
   необходим нам для того, что бы для каждого нового метода с похожей реализацией не выдумывать нового имени. Примеры:
   print(int a), print(int a, String b) - методы, которые имеют одинаковое наименование, но количество входных
   параметров отличается -> скорее всего, реализация данным методов тоже будет отличаться, это и есть перегрузка.  
2. **Переопределение** (**Overriding**)- механизм, который позволяет нам переписать реализацию унаследованного метода.

К примеру, мы унаследовались от интерфейса или абстрактного класса у которого есть метод print(). В нашем классе мы тоже
хотим иметь метод print(), но реализация данного метода в суперклассе нас не устраивает. Мы можем просто взять и
объявить точно такой же метод с таким же количеством параметров в нашем классе и просто переписать логику, возможно,
используя логику метода в суперклассе используя ключевое слово super.

| Критерий             | Перегрузка                           | Переопределение                   |
|----------------------|--------------------------------------|-----------------------------------|
| **Место**            | Один класс                           | Родительский и дочерний класс     |
| **Сигнатура**        | Разная (разные параметры)            | Та же самая                       |
| **Цель**             | Удобство вызова похожих операций     | Изменение поведения у наследников |
| **Время выбора**     | Компиляция (раннее связывание)       | Выполнение (позднее связывание)   |
| **Возвращаемый тип** | Может отличаться                     | Должен быть совместим             |
| **Доступ**           | Любой модификатор                    | Нельзя сужать доступ              |
| **Исключения**       | Любые                                | Нельзя добавлять новые            |
| **Аннотация**        | Не требуется                         | @Override (рекомендуется)         |
| **Простая аналогия** | Разные инструменты с одним названием | Новая логика на той же должности  |

**Перегрузка** → компилятор решает.  
**Переопределение** → JVM решает.
